# SANDBOX Storage Rules

## Default behavior

Use `caosos/SANDBOX` as the default working repository for project-related creations unless Michael says otherwise.

## Put here

- Markdown docs
- Landing page copy
- Service scripts
- Call sheets in Markdown or CSV
- SOPs
- Checklists
- Project plans
- Simple code and prototypes
- Templates
- Sanitized research notes

## Be careful with

- XLSX files
- PDFs
- Images
- Large binaries
- Raw exports

Prefer Markdown, CSV, JSON, or plain text unless there is a strong reason to store a binary file.

## Do not put private material here

This is a public repository. Keep private personal, company, resident, customer, personnel, and medical information out of it. Store sanitized templates instead.

## Naming

Use clear folder names and date files when helpful:

```text
YYYY-MM-DD-topic.md
YYYY-MM-DD-call-sheet.csv
project-decision-log.md
```

## Repo discipline

If a project becomes real enough to stand alone, split it into its own repository later.
